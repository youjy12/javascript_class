검색엔진최적화(SEO) - meta

메타 태그(meta tag)는 웹 서버와 웹 브라우저간에 상호 교환되는 정보를 정의하는데 사용하며, 웹페이지가 담고 있는 컨텐츠가 아닌 웹페이지 자체의 정보를 명시하기 위한 목적으로 사용되는 HTML 태그를 의미합니다.

이러한 웹페이지의 메타 정보는 주로 HTML의 meta요소를 통해 마크업을 하며, HTML 문서 내에서 head 요소 아래 배치되므로 일반 유저가 보게되는 웹페이지의 디자인에는 전혀 영향을 미치지 않고, 문서가 어떤 내용을 담고 있고, 문서의 키워드는 무엇이며, 누가 만들었는지 등의 문서 자체의 특성을 담고 있습니다.

반면에 검색엔진(네이버, 구글, 다음, 네이트...)과 같은 기계들이 웹페이지를 읽어야 할 때는 메타 태그의 내용들이 해당 서비스에서 어떻게 표시될지를 결정하는 매우 핵심적인 요소가 됩니다.

SEO 태그 - title

검색엔진은 주기적으로 웹사이트를 방문하여 각 웹페이지가 어떤 콘텐츠를 제공하고 있는지를 파악하고, 수집하여 데이터베이스에 저장을 하는데, 점점 지능적(AI)으로 진화하고 있습니다. 또한 어느 정도까지는 스스로 웹페이지의 구조와 내용을 파악할 수 있습니다. 그럼에도 불구하고 메타태그를 통해 웹페이지에 대한 정보를 좀 더 주도적으로 명시한다면 SEO(검색엔진 최적화)측면에서 유리합니다.

예) 메인페이지인 경우 => 스타벅스에 오신것을 환영합니다.

예) 서브페이지 => 스타벅스 - 오시는길 입니다.

SEO 태그 - meta

웹사이트이 제목만 예외적으로 title 요소를 사용하고, 그 밖에 다른 웹페이지의 메타 정보는 meta 요소를 사용하여 표현합니다.

meta요소를 사용할 때는 name속성을 통해 메타정보의 이름을 명시하고 content속성을 통해 메타 정보의 이름과 내용을 명시합니다.

meta 태그의 3가지 속성

1. http-equiv="항목명" : 웹 브라우저가 서버에 명령을 내리는 속성으로 name 속성을 대신하여 사용될 수 있으며, HTML 문서가 응답 헤더와 함께 웹 서버로부터 웹 브라우저에 전송되었을 때에만 의미

2. content="정보값" : meta정보의 내용을 지정합니다.

3. name="정보이름" : 몇 개의 meta정보의 이름을 지정할 수 있으며 지정하지 않으면 http-equiv와 같은 기능을 합니다.

meta 태그의 종류

meta name="keywords" content="여성의류, 웹표준" : 검색엔진에 의해 검색되는 단어를 지정

meta name="Description" content="신상의류, 자신의 스타일에 맞는 의상 추천" : 검색 결과에 표시되는 문자를 지정

meta name="Robots" content="noindex, nofollow" : 검색 로봇 제어

  • All(기본값) : 'index, follow'
  • None = 'noindex, nofollow'
  • Index : 그 페이지를 수집 대상으로 합니다.
  • Follow : 그 페이지를 포함해 링크가 걸린 곳을 수집 대상으로 합니다.
  • Noindex : 그 페이지를 수집대상에서 제외
  • Nofollow : 그 페이지를 포함하여 링크가 걸린 곳을 수집 대상으로 하지 않습니다.

meta http-equiv="Content-Type" content="text/html; charset=utf-8" : 문자 코드의 종류를 설정

meta name="Date" content="2023-01-19T07:45:37+09:00" : 작성날짜(제작일)

meta http-equiv="X-UA-Compatible" content="IE=edge" : 브라우저 호환성을 지정

meta http-equiv="Subject" content="웹 표준을 위한 사이트" : 홈페이지 주제

meta http-equiv="Title" content="웹 표준" : 홈페이지의 제목

meta http-equiv="Author" content="name" : 작성자

meta http-equiv="Other Agent" content="name" : 책임자

meta http-equiv="Distribution" content="name" : 배포자

meta http-equiv="Copyright" content="copyright@dothome" : 저작권

meta http-equiv="Build" content="date" : 제작일

meta http-equiv="Last-Modified" content="Mon,20 Jul 2016 19:30:30" : 최종수정일

meta http-equiv="Refresh" content="60" : 지정한 시간마다 새로고침

meta http-equiv="Refresh" content="5;url=주소" : 입력한 주소로 지정시간후 자동연결

meta http-equiv="Page-Enter" content="revealtrans(Duration=1,Transition=12)" : 페이지 장면전환 효과

SEO태그 - Open Graph

요즘에는 SNS와 메신저를 통해 웹페이지의 링크를 공유하는 일이 잦아지면서 메타 태그가 좀 더 넓은 분야에서 활용되고 있습니다. 대표적인 사례로 페이스북이나 카카오톡, 슬랙(Slack)과 같은 앱에서 웹페이지의 링크만 입력하더라도 해당 웹페이지의 콘텐츠의 미리보기를 카드 형태로 나타나는 것을 들 수 있습니다.

일반적으로 이러한 서비스들은 Open Graph 프로토콜이라고 불리는 업계 표준을 따라서 웹페이지 콘텐츠 미리보기를 지원하고 있습니다. 따라서 우리도 Open Graph 프로토콜에서 정의하고 있는 메타 태그를 적절히 사용하면 콘텐츠 미리보기에 원하는 내용을 표시되도록 어느 정도 제어를 할 수 있습니다.

Open Graph 태그의 이름은 og:로 시작

meta name="og:site_name" content="사이트명" : 사이트 제목

meta name="og:title" content="페이지 제목" : 웹페이지의 제목

meta name="og:description" content="페이지 설명" : 웹페이지의 상세 설명

meta name="og:type" content="webpage" : 웹페이지의 유형

meta name="og:url" content="http://design2010.dothome.co.kr" : 웹페이지의 주소

meta name="og:image" content="http://이미지 경로" : 웹페이지의 썸네일 이미지 주소
